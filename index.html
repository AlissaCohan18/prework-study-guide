HTML

  • The head element contains information about the webpage/li>
  • The body element represents the visible content shown to the user
  • ul stands for unordered lists
  • li stands for (bulleted) lists

CSS

  • CSS is for formatting the content
  • Margin indicates how much space we want around the outside of an element. ie-space outside of box/border
  • Padding indicates how much space we want around the content inside an element. space between content and box/border
  • Use a "," between two selectors to apply rule to multiple. Example: "h1, h2" text-align center
  • The "." preceding "card" designates this as a class selector. Any HTML element assigned to the class, card, will gain the property assignments

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches to main branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A (adds the changed files in our local working directory to the staging area of Git)
  • git commit -m "added code to HTML file" (commits staged changeds repository; with comments noted by -m in ""
  • git pull origin main (ensures current branch is up to date before opening a PR
  • git push origin XYZ [Pushes changes from XYZ (feature) branch to github where it's ready for review]

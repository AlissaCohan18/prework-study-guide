HTML

  • li stands for (bulleted) lists
  • Attributes (red text with in my VS Code scheme) include: class, src, href, id

CSS

  • CSS is for formatting the content
  • Margin indicates how much space we want around the outside of an element. ie-space outside of box/border
  • Padding indicates how much space we want around the content inside an element. space between content and box/border
  • Use a "," between two selectors to apply rule to multiple. Ex/ "h1, h2" text-align center
  • The "." preceding "card" designates this as a class selector. Any HTML element assigned to the class, card, will gain the property assignments
  • html: class=lesson === css: .lesson
    html: <h1> === css: h1
    html: id=“github” === css: #github
  • Can assign characteristics to Mult by using a , Ex/ header, footer { color: red }
  • the * is a wild card to select/apply to everything.
    Ex/ * { font-family: "Lucida Sans Unicode", "Lucida Grande", sans-serif; }
    html: <p> blah blah blah </p>
    css: the below will apply to everything with a p-tag
    p { font-size: 60px; line-height: 200px; }

Git

  • note: navigate to the directory we want to work within, then...
  • git status: checks what branch we are currently on
  • git checkout main: switches to main branch
  • git checkout featureXYZ: switches to the featureXYZ branch
  • git pull origin main (to pull the remote main branch into the local main branch so that local and remote branches are in sync)
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A (adds the changed files in our local working directory to the staging area of Git)
  • git commit -m "added code to HTML file" (commits staged changeds repository; with comments noted by -m in ""
  • git pull origin main (ensures current branch is up to date before opening a PR)
  • git push origin XYZ [Pushes changes from XYZ (feature) branch to github where it's ready for review]
  • git init: use when you have existing content in a local directory/computer and want to create a new repo on git
  • git log: returns details regarding commits
  • git remote -v: Checks if there is a git repo associated with dir
  • To add SSH key for the session so won't be prompted for password again, in terminal-> ssh-add
